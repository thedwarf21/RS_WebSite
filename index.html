Le jeu Asteroids Harvest

Asteroids Harvest est un jeu HTML5 open source développé from scratch par nos soins à titre de démonstration.

Il s'agit d'une revisite du jeu Asteroids édité par Atari en 1979 et maintes fois cloné depuis lors. Asteroids Harvest propose également :

  • La possibilité d'améliorer le vaisseau, via un magasin s'ouvrant avant chaque niveau.
  • Lorsqu'un astéroïde est touché par un tir, sa trajectoire est modifiée selon l'angle et la puissance du tir.
  • Un système de sauvegarde par cookie.
  • Affichage d'un HUD, sur les appareils mobiles.
  • Adaptation aux dimensions d'affichage, en temps réel.
  • Possibilité d'afficher les hitbox.
  • Sensibilité radiale paramétrable.

Problèmatique

Solution apportée

Il s'agit d'un jeu de type temps réel, ce qui implique une éxécution périodique des fonctions permettant :

  • Le déplacement de tous les objets.
  • La lecture et la prise en compte des commandes du joueur.
  • Les tests de collision.

Pour un affichage en 20 FPS, la période est de 50 ms.

Sur navigateur web, les ressources sont extrêmement limitées.

Le code est optimisé au maximum, de manière à ce que chacune des opérations effectuée périodiquement, s'exécute en temps et en heure, afin d'éviter que l'animation ne saccade.

Le magasin, les éléments du jeu (vaisseau, astéroïdes, bonus, tirs), les fonctions liées à l'animation temps réel, l'adaptation des mécaniques du jeu en fonction des dimensions de la zone d'affichage, ou encore la gestion des hitbox, sont des problématiques bien distinctes.

Afin que chacun de ces aspects puisse être facilement mis à jour indépendamment les uns des autres, le code a été structuré en classes gérant chacune un aspect du jeu.

Le ratio de la zone d'affichage (fenêtre du navigateur web) peut varier, d'un appareil à l'autre, et peut même être modifié en cours de partie, par le joueur.

De plus, afin que les angles des trajectoires restent cohérents avec l'angle d'orientation du vaisseau, travailler avec des coordonnées absolues positionnelles est obligatoire (deltaX = cos(angle) x vitesse; deltaY = sin(angle) x vitesse). Malheureusement, cela implique, en théorie, d'utiliser des dimensions fixes, pour la zone d'affichage du jeu.

Utilisation de coordonnées positionnelles virtuelles, qui sont ensuite traduites en pourcentage des dimensions d'affichage.

Afin de prendre en compte l'inconstance du ratio (largeur / hauteur) de la zone d'affichage, la classe gérant cet aspect utilise un seul axe de référence.

Sur l'axe de référence, les coordonnées réeeles en pourcentage sont calculées sur la base d'une dimension virtuelle, paramétrée à l'aide d'une constante.

Sur l'autre axe, les coordonnées réelles en pourcentage sont calculées en fonction du ratio de la zone d'afficahge, celui-ci étant mis à jour lors de chaque redimensionnement.

Les tailles pour chaque élément graphique du jeu (polices de caractère, vaisseau, bonus, astéroïdes, etc.) sont toutes exprimées en pourcentage des dimension de la zone d'affichage, sur l'axe de référence.

Nous avons choisi l'axe vertical comme axe de référence, afin que la superficie vituelle de la zone d'affichage soit proportionnelle au ratio réel. Les dimensions sont donc exprimées en vh.

Le HUD doit s'afficher uniquement sur les appareils mobiles, afin d'éviter de polluer l'affichage, lorsque le jeu est utilisé sur un écran d'ordinateur. En effet, les contrôles du jeu supportant une utilisation du clavier (touches fléchées, barre espace et touche 'P'), le HUD devient inutile sur ordinateur.

Utilisation des media queries CSS pour masquer le HUD.

Le code source et le jeu en lui-même, sont accessibles publiquement sur la plateforme GitHub.

Le jeu Démineur

MineSweeper est lui aussi, un jeu HTML5 open source développé from scratch par nos soins à titre de démonstration.

Il s'agit d'une revisite du jeu Démineur, jouable gratuitement sur Windows ©, même dans ses versions les plus anciennes. Particularités de notre version web :

  • La marque ? n'a pas été intégré, dans les états des cases, car jugée inutile.
  • Aucun visuel graphique (fichier image) n'a été utilisé, excepté pour l'icône.

N'utilisant aucune image, le jeu dans sa totalité n'occupe que 50,4 ko d'espace disque, sur le serveur.

Le code source et le jeu en lui-même, sont accessibles publiquement sur la plateforme GitHub.

Applications hybrides en web natif

Les projets AsteroidsHarvest et MineSweeper ont tous deux été développés, de manière à pouvoir être utilisés comme des applications autonomes, sur les appareil mobile.

Pour ce faire, il vous suffira de suivre les étapes suivantes :

  • Ouvrir le jeu, sur le navigateur web de votre appareil mobile.
  • Appuyer sur le bouton d'options (à droite, dans la barre d'outils du navigateur).
  • Choisir "Ajouter à l'écran d'accueil".
  • Ouvrir le jeu depuis l'icône ainsi créée sur l'écran d'accueil de votre appareil mobile.

Vous constaterez que lorsque vous ouvrez le jeu de cette manière :

  • Vous n'êtes plus pollué par la barre d'outils du navigateur, contenant la barre d'adresse.
  • L'affichage est forcé en paysage (non garanti sur d'autres navigateurs que Chrome Mobile, à ce jour)
